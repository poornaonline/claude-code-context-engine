# Examples

## Todo API — Framework Output Reference

The `todo-app/` directory contains sample output from running the Claude Code Context Engine framework on a simple Todo REST API project.

**What this demonstrates:**
- A minimal PRD that captures just enough for AI-assisted development
- A CLAUDE.md tattoo file with navigation pointers (not content duplication)
- Progressive disclosure specs that load only what's needed per task
- Task cards with explicit context-loading instructions
- Bootstrap protocol with decision tree and subagent templates
- Session handoff and crash recovery patterns

**How to use it:**
1. Browse the files to understand what "good" framework output looks like
2. Compare against your own project's generated files
3. Use it as a template when customizing the framework for your needs

**Tech stack of the example project:** Node.js 22.x, Express 5.x, SQLite (better-sqlite3), Vitest

**Note:** This is reference output only. The `todo-app/` directory does not contain runnable application code — just the framework documentation layer.
